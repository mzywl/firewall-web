- 分析策略是否需要NAT转换
- 判断源IP和目的IP所属区域（internal/external）
- 根据跨区域规则生成NAT类型（SNAT/DNAT/BOTH）

**核心方法**：
```python
def analyze_policy(source_ip, dest_ip, firewall) -> Dict
```

## 二、核心功能实现

### 2.1 防火墙匹配逻辑 ✅
- **数据来源**: `Firewall`表的`internal_protected_ips`和`external_protected_ips`字段
- **匹配规则**: 根据目的IP的第一个IP地址匹配防火墙管理范围
- **实现**: 使用`FirewallMatcher.match_by_ip()`方法
- **结果**: 匹配成功返回`firewall_id`，失败返回`None`

### 2.2 NAT转换逻辑 ✅
- **区域判断**: 根据IP段判断源IP和目的IP所属区域（internal/external）
- **NAT类型判断**:
  - 内部→外部 = SNAT（出向）
  - 外部→内部 = DNAT（入向）
  - 其他跨区域 = BOTH（双向NAT）
- **地址池读取**:
  - SNAT: `firewall.outbound_snat_pool`
  - DNAT: `firewall.inbound_dnat_pool`
- **警告生成**: 地址池缺失时生成警告信息

### 2.3 NAT策略生成 ✅
- **SNAT策略**: 源IP替换为SNAT地址池，标记[SNAT]
- **DNAT策略**: 目的IP替换为DNAT地址池，标记[DNAT]
- **双向NAT**: 生成两条策略（SNAT + DNAT）
- **缺失处理**: 地址池缺失时显示"[需要配置SNAT地址]"

### 2.4 页面展示 ✅
- **防火墙分组**: 按`firewall_id`分组，每个防火墙一个卡片
- **策略表格**: 7列（源区域、源IP、目的区域、目的IP、服务/端口、动作、NAT）
- **NAT标记**: 
  - 原始策略行：显示NAT类型图标
  - NAT转换后策略行：蓝色背景 + [SNAT]/[DNAT]标签
- **异常提示**: 页面顶部黄色卡片 + 策略行下方警告

### 2.5 自动执行 ✅
- **默认状态**: `auto_push=1`的防火墙默认勾选
- **状态保存**: 保存到`localStorage`，传递给推送页面
- **交互**: 复选框可切换
