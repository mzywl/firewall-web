# Excel 解析模块更新说明

## Issue #3: 表格格式化优化

### 更新内容

#### 1. Sheet 读取优先级
- **优先读取名称包含"网络策略"的 sheet**
- 如果没有找到，则使用激活的 sheet

#### 2. 第一次格式化：查找真正的表头
- 从第一行开始（包括表头）查找包含以下 3 个关键字段的行：
  - 源IP
  - 目的IP
  - 目的端口
- 这 3 个字段会分布在不同的列
- 找到后，将这一行作为新的表头
- 展示这之后的内容

#### 3. 第二次格式化：删除示例策略
- 删除示例策略行
- 判断条件：如果同时满足以下条件，则认为是示例策略：
  - "源端系统-环境-用途" 包含 "示例"
  - "目的端系统-环境-用途" 包含 "示例"

#### 4. 数据格式化
- 格式化 IP 地址
- 格式化端口
- 格式化日期
- 去除首尾空格

### 使用示例

```python
from app.core.excel_parser import ExcelParser

# 解析 Excel 文件
parser = ExcelParser("test.xlsx")
result = parser.parse()

# 返回结果
{
    "headers": ["源IP", "目的IP", "目的端口", ...],
    "data": [
        {"源IP": "192.168.1.0/24", "目的IP": "10.0.0.1", ...},
        ...
    ],
    "total_rows": 10,
    "header_row": 3  # 真正的表头在第 3 行
}
```

### 测试场景

1. **场景1：标准格式**
   - 第一行就是表头
   - 包含所有关键字段
   - 无示例策略

2. **场景2：表头在中间**
   - 前几行是说明文字
   - 第 N 行才是真正的表头
   - 包含关键字段

3. **场景3：包含示例策略**
   - 表头正常
   - 数据中包含示例策略行
   - 需要自动过滤

4. **场景4：多个 Sheet**
   - 包含多个 sheet
   - 其中一个名称包含"网络策略"
   - 优先读取该 sheet

### 关键字段说明

必须包含以下 3 个关键字段才能识别为表头：
- **源IP**：源 IP 地址字段
- **目的IP**：目的 IP 地址字段
- **目的端口**：目的端口字段

这些字段可以在任意列，只要行中包含这 3 个字段即可。

### 示例策略过滤

示例策略的判断条件：
- **源端系统-环境-用途** 包含 "示例"
- **目的端系统-环境-用途** 包含 "示例"

只有同时满足这两个条件，才会被过滤掉。

### 兼容性

- 向后兼容：如果 Excel 文件格式标准（第一行就是表头），解析逻辑不变
- 新功能：自动识别非标准格式的 Excel 文件
- 智能过滤：自动删除示例策略，减少手动清理工作
